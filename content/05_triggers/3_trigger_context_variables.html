30-Second Interview Answer

Trigger context variables are system-provided variables that give information about the trigger‚Äôs

execution context, such as the records being processed and the type of operation. Examples include

Trigger.new, Trigger.old, Trigger.newMap, Trigger.oldMap, and boolean flags like Trigger.isInsert or

  • Trigger.isUpdate.
2-Minute Technical Explanation
  • When a trigger runs, Salesforce automatically provides context variables that help us:
  • Access records
  • Identify operation type
  • Compare old vs new values
  • Handle bulk processing properly
  • Main context variables:
Record Collections
  • Trigger.new
  • Trigger.old
  • Trigger.newMap
  • Trigger.oldMap
Boolean Flags
  • Trigger.isInsert
  • Trigger.isUpdate
  • Trigger.isDelete
  • Trigger.isBefore
  • Trigger.isAfter
  • Trigger.isUndelete
  • They are critical for writing correct and bulk-safe trigger logic.

üî¨DEEP TECHNICAL BREAKDOWN
Trigger.new
  • List of new versions of records.
  • Available in:
  • before insert
  • before update
  • after insert
  • after update
  • after undelete
Example:
for(Account acc : Trigger.new){
System.debug(acc.Name);
}
Trigger.old
  • List of old versions.
  • Available in:
  • before update
  • before delete
  • after update
  • after delete
  • Used for comparison.
Trigger.newMap
  • Map
  • Available in:
  • before update
  • after update
  • after insert
  • after undelete
Example:
Account acc = Trigger.newMap.get(accId);
  • Faster lookup than looping.
Trigger.oldMap
  • Map
  • Available in:
  • before update
  • after update
  • before delete
  • after delete
  • Used to compare previous values.
Trigger.isInsert
  • True when event is insert.
Trigger.isUpdate
  • True when update.
Trigger.isDelete
  • True when delete.
Trigger.isBefore / isAfter
  • Helps differentiate logic.
Trigger.isUndelete
  • Used in after undelete event.
