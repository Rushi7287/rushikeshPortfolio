5.a Trigger Syntax

üé§INTERVIEW SECTION
30-Second Interview Answer

A Trigger in Salesforce is Apex code that executes automatically in response to DML events like insert,

update, delete, or undelete on an object. The basic syntax defines the object and the event context.

  • Triggers operate in bulk and must follow governor limit best practices.
2-Minute Technical Explanation
  • Trigger Syntax:
trigger TriggerName on ObjectName (event1, event2) {
  • // logic
}
Example:
trigger AccountTrigger on Account (before insert, before update) {
for(Account acc : Trigger.new){
acc.Description = 'Updated by Trigger';
}
}
  • Trigger Components:
  • Trigger Name
  • Object
  • Events
  • Trigger Context Variables
  • Business Logic
  • Triggers can run on:
  • before insert
  • before update
  • before delete
  • after insert
  • after update
  • after delete
  • after undelete
Real-Time Scenario
Requirement:
  • When Opportunity is Closed Won:

Trigger:
trigger OpportunityTrigger on Opportunity (after update) {
List<Project__c> projects = new List<Project__c>();

insert projects;
}

Full Trigger Syntax

  • after insert,
  • after update,

  • after undelete
) {
  • // logic
}
Before Trigger
  • Used to modify field values.
  • No extra DML required.
Example:
trigger AccountBefore on Account (before insert) {
for(Account acc : Trigger.new){
acc.Name = acc.Name + ' - Updated';
}
}
After Trigger
  • Used when record ID required.
  • Used to create related records.
Example:
trigger AccountAfter on Account (after insert) {
List<Contact> contacts = new List<Contact>();
}
Trigger Context Variables
Available:
  • Trigger.new
  • Trigger.old
  • Trigger.newMap
  • Trigger.oldMap
  • Trigger.isInsert
  • Trigger.isUpdate
  • Trigger.isDelete
  • Trigger.isAfter
  • Trigger.isBefore
Bulk Trigger Example
trigger AccountBulk on Account (after update) {
Set<Id> accIds = new Set<Id>();
for(Account acc : Trigger.new){
accIds.add(acc.Id);
}
  • List conList = [
  • SELECT Id FROM Contact
  • WHERE AccountId IN :accIds
];
update conList;
}
  • Bulk safe.

‚Üí Calls handler class.
Trigger Framework Pattern
Trigger:
trigger AccountTrigger on Account (before insert, before update) {
AccountHandler.handle(Trigger.new);
}
  • Handler Class:
public class AccountHandler {
public static void handle(List<Account> accList){
  • // logic
}
}
  • Clean architecture.
Governor Limit Awareness
Trigger:
  • Runs in same transaction.
  • Shares governor limits.
  • Cannot reset limits.
  • Limited to 100 SOQL / 150 DML.
Recursive Trigger
  • Update inside after update ‚Üí infinite loop.
  • Use static flag.
Mixed DML
  • Updating User and Account together ‚Üí error.
Large Data Volume
  • Mass update ‚Üí CPU limit risk.
  • Use Batch Apex.
Debugging Trigger Issues
  • 1Enable Debug Logs.
  • 2Check execution order.
  • 3Check recursion.
  • 4Check governor limit usage.
  • 5Check validation rules interaction.
When NOT to Use Trigger
  • Avoid trigger if:
  • Simple field update ‚Üí Use Flow.
  • Simple validation ‚Üí Use Validation Rule.
  • Simple email ‚Üí Use Flow.
  • Use Trigger when:
  • Complex logic.
  • Performance control.
  • Cross-object updates.
  • Advanced integration handling.
