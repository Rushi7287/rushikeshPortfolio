üé§INTERVIEW SECTION
30-Second Interview Answer
Mixed DML error occurs when you attempt to perform DML operations on setup objects (like User,
Profile, PermissionSetAssignment) and non-setup objects (like Account, Opportunity) in the same

transaction. Salesforce blocks this to maintain data integrity. The solution is to separate operations

  • using asynchronous Apex like Queueable or Future.
2-Minute Technical Explanation
  • Salesforce classifies objects into:
Setup Objects
  • User
  • Profile
  • PermissionSet
  • PermissionSetAssignment
  • Group
  • Queue
Non-Setup Objects
  • Account
  • Contact
  • Opportunity
  • Custom Objects
  • If you write:
insert new Account(Name='Test');
insert new User(...);
  • You get:
  • Mixed DML Operation Error
Because:
  • Setup objects affect org-level access control
  • Non-setup objects affect business data
  • Both cannot be committed in same transaction
Solution:
  • Separate them using async processing.

  • Correct approach:
  • Trigger ‚Üí
  • Insert Account ‚Üí

  • Queueable inserts User.
  • Now separate transactions.

üî¨DEEP TECHNICAL BREAKDOWN
Why Mixed DML Exists
Because:
  • Setup object changes impact:
  • Security model
  • Sharing
  • Access hierarchy
  • Salesforce restricts mixing them with business data changes in same transaction.
Prevents:
  • Partial security update
  • Inconsistent sharing state
Example Causing Error
Account acc = new Account(Name='Test');
insert acc;
User u = new User(
Username='test@test.com',
Email='test@test.com',
LastName='Test',
Alias='tst',
TimeZoneSidKey='Asia/Kolkata',
LocaleSidKey='en_US',
EmailEncodingKey='UTF-8',
ProfileId=profileId,
LanguageLocaleKey='en_US'
);
insert u;
Throws:
  • DML operation on setup object is not permitted after DML on non-setup object
Correct Solution ‚Äì Queueable
Trigger:
System.enqueueJob(new CreateUserQueueable(acc.Id));
Queueable:
public void execute(QueueableContext context){
insert new User(...);
}
Now:
  • Account committed
  • New transaction creates User
  • No Mixed DML
Correct Solution ‚Äì Future
  • @future
public static void createUserAsync(Id accId){
insert new User(...);
}
  • Works but Queueable preferred.
Fix in Test Class
  • Test example:
User u = new User(...);
insert u;
System.runAs(u){
insert new Account(Name='Test');
}
  • runAs creates separate execution context.
Common Setup Objects Causing Error
  • User
  • Group
  • Queue
  • PermissionSetAssignment
  • Territory
  • Role
Remember: Mostly security-related objects.
Governor Limit Impact
  • Mixed DML:
  • Happens before commit
  • Entire transaction fails
  • No partial success
  • Must design around it.
