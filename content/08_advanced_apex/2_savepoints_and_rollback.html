30-Second Interview Answer

Savepoints allow you to mark a point in a transaction to which you can later roll back if an error occurs.

Using Database.setSavepoint() and Database.rollback(), you can undo DML operations within the same
  • transaction without affecting prior committed data.
2-Minute Technical Explanation
  • By default:
  • If an exception occurs ‚Üí
  • Entire transaction rolls back.
  • But sometimes you want:
  • Partial control inside same transaction
  • Multi-step logic with validation
  • Manual undo mechanism
Syntax:
Savepoint sp = Database.setSavepoint();
  • // DML operations
Database.rollback(sp);
Important:
  • Works only within same transaction
  • Cannot rollback after commit
  • Does not reset governor limits

üî¨DEEP TECHNICAL BREAKDOWN
Why Savepoint Needed
  • Default DML behavior:
  • If exception thrown ‚Üí entire transaction rolls back automatically.
  • But sometimes you want:
  • Control specific part of transaction.
Basic Savepoint Example
Savepoint sp = Database.setSavepoint();
try{
insert acc;
insert contactList;
}
catch(Exception e){
Database.rollback(sp);
}
  • If Contact insert fails:
  • Account also rolled back.
Multiple Savepoints Example
Savepoint sp1 = Database.setSavepoint();
insert acc;
Savepoint sp2 = Database.setSavepoint();
insert contact;
Database.rollback(sp2);
  • This rolls back only to sp2.
  • Account remains.
Savepoint Limit
Maximum:
  • 5 savepoints per transaction.
  • Exceed ‚Üí LimitException.
Rollback Only Affects DML
  • It reverses:
  • Insert
  • Update
  • Delete
  • Upsert
  • It does NOT:
  • Undo callouts
  • Reset governor limits
  • Undo email sent
Rollback Does Not Clear Variables
  • Only database state changes.
Governor Limit Behavior
Rollback:
  • Does NOT reset SOQL count
  • Does NOT reset DML count
  • Does NOT reset CPU time
  • Limits continue accumulating.

Savepoint in Async
  • Works, but only within that async transaction.
Rollback After Commit
  • Impossible.
Nested Try-Catch
  • Use carefully to avoid partial inconsistent state.
When NOT to Use Savepoint
  • Avoid if:
  • Using Batch for partial processing
  • Using Database methods for partial success
  • No multi-step dependency
  • Logic simple
