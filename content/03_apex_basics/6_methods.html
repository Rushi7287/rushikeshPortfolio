30-Second Interview Answer

Methods in Apex are reusable blocks of code defined inside a class that perform specific actions. They

can accept parameters, return values, and have access modifiers like public, private, static, or global.

Methods help organize business logic, promote reusability, and maintain clean architecture.

2-Minute Technical Explanation
  • A method in Apex consists of:
[access modifier] [static] returnType methodName(parameters) {
  • // logic
}
Example:
public static Integer add(Integer a, Integer b) {
return a + b;
}
  • Key components:
  • Access Modifier ‚Üí who can access
  • Static/Instance ‚Üí belongs to class or object
  • Return Type ‚Üí what method returns
  • Parameters ‚Üí input values
  • Method Body ‚Üí logic
Real-Time Scenario
Requirement:
  • When Opportunity closes:
‚Üí Calculate commission
‚Üí Update Account rating
‚Üí Send notification
  • Instead of writing all logic in trigger,
  • We create reusable methods:
calculateCommission()
updateAccount()
sendNotification()
  • Improves maintainability and testing.
Tricky Interview Questions
  • 1Can method return multiple values?
‚ùå Directly no (use wrapper class or Map).
  • 2Can method be overloaded?
Yes.
  • 3Can static method access instance variables?
No.
  • 4Can instance method access static variables?
Yes.
  • 5What happens if no return statement in non-void method?
Senior-Level Discussion Points
  • Method overloading
  • Recursion handling
  • Dependency injection
  • Bulk-safe method design
  • Returning wrapper classes
  • Avoiding mixed DML
  • With sharing vs without sharing impact

Basic Method Syntax
public class MyClass {
public static void greet() {
System.debug('Hello');
}
}
Call:
MyClass.greet();
Method with Return Type
public static Integer multiply(Integer a, Integer b) {
return a * b;
}
Method with Parameters
public static String getGreeting(String name) {
return 'Hello ' + name;
}
Void Method
public static void updateAccount(Account acc) {
acc.Rating = 'Hot';
update acc;
}
  • Returns nothing.
Method Overloading
  • Same method name,
  • Different parameters.
public static Integer calculate(Integer a, Integer b) {
return a + b;
}
public static Decimal calculate(Decimal a, Decimal b) {
return a + b;
}
  • Apex decides based on parameter type.
üìå Bulk-Safe Method (Important)
Bad ‚ùå
public static void updateAccounts(Account acc){
update acc;
}
  • Called inside loop ‚Üí DML limit error.
Good ‚úÖ
public static void updateAccounts(List<Account> accList){
update accList;
}
  • Always design methods for bulk.
üìå Returning Multiple Values (Wrapper Class)
public class ResultWrapper {
public Decimal commission;
public Boolean isEligible;
}
Method:
public static ResultWrapper calculate(){
ResultWrapper rw = new ResultWrapper();
rw.commission = 100;
rw.isEligible = true;
return rw;
}
Recursive Method Example
public static Integer factorial(Integer n){
if(n == 1){
return 1;
}
return n * factorial(n-1);
}
  • ‚ö† Risk of stack overflow if not controlled.
Methods in Trigger Framework
Trigger:
trigger AccountTrigger on Account (after update) {
AccountService.process(Trigger.new);
}
  • Service Class:
public class AccountService {
public static void process(List<Account> accList){
  • // logic
}
}
  • Best practice: Keep trigger thin.
Performance Considerations
Avoid:
  • SOQL inside method loop
  • Repeated queries
  • Unnecessary object instantiation
  • Large return objects
Prefer:
  • Using Maps for lookup
  • Using Set for Id collection
  • Bulk DML
  • Caching values
Governor Limit Awareness
  • Each method:
  • Runs within same transaction
  • Shares governor limits
  • Does NOT reset limits
  • Async method:
  • New transaction ‚Üí new limits.
Calling Method from Future
  • Must be static and annotated @future.
Callout Method
  • Requires callout=true in future/queueable.
Method Visibility in LWC
  • Must be:
  • @AuraEnabled
public static void methodName(){}
Debugging Method Issues
  • If method not working:
  • 1Check access modifier.
  • 2Check return type.
  • 3Check null handling.
  • 4Check bulk handling.
  • 5Check limit usage.
When NOT to Create Separate Method
  • Avoid creating too many tiny methods that:
  • Increase complexity.
  • Break readability.
  • Over-engineer simple logic.
  • Balance modularity.
Interview Power Line
  • If interviewer asks:
  • ‚ÄúWhat makes a good Apex method?‚Äù
Answer:

A good Apex method is bulk-safe, reusable, properly scoped with correct access modifier, limit-aware,

  • and follows single responsibility principle.

4.a SOQL (Salesforce Object Query Language)
üé§INTERVIEW SECTION
30-Second Interview Answer
SOQL (Salesforce Object Query Language) is used to retrieve records from Salesforce objects. It is similar

to SQL but designed for Salesforce‚Äôs object-based architecture. SOQL allows querying fields,

relationships, and applying filters, and it is subject to governor limits such as 100 queries per

  • synchronous transaction.
2-Minute Technical Explanation
  • SOQL is used to fetch data from:
  • Standard Objects
  • Custom Objects
  • Custom Metadata
Big Objects (limited)
  • Basic structure:
  • SELECT field1, field2
  • FROM ObjectName
  • WHERE condition
  • ORDER BY field
  • LIMIT number
Example:
  • SELECT Id, Name
  • FROM Account
  • WHERE Industry = 'IT'
  • ORDER BY CreatedDate DESC
  • LIMIT 10
  • Key points:
Must specify fields (no SELECT *)
  • Supports relationship queries
  • Case-insensitive keywords
  • Enforced by governor limits
Real-Time Scenario
Requirement:
  • When Opportunity closes:
  • Fetch related Account and update its Rating.
Steps:
  • 1Collect Account IDs in Set.
  • 2Query Accounts using IN clause.
  • 3Update in bulk.
  • SOQL is core for almost every Apex logic.
Tricky Interview Questions
  • 1Why can't we use SELECT * in SOQL?
‚Üí Performance and multitenant optimization.
  • 2What is SOQL query limit in sync Apex?
‚Üí 100.
  • 3What is limit in async?
‚Üí 200.
  • 4What is selective query?
‚Üí Query using indexed fields and returning <10% records.
  • 5What happens if query returns more than 50,000 rows?
‚Üí QueryException.
Senior-Level Discussion Points
  • Query selectivity
  • Query Plan Tool
  • Indexing
  • Skinny tables
  • Bind variables
  • Query optimization strategy
  • FOR UPDATE clause
  • Aggregate SOQL limits

Basic SOQL Syntax

[SELECT Id, Name FROM Account];

Filtering Records
  • SELECT Id, Name
  • FROM Account

üìå Using Bind Variables (Important)
String industry = 'IT';
  • List accList = [
  • SELECT Id, Name
  • FROM Account
  • WHERE Industry = :industry
];
  • Prevents SOQL injection.
üìå IN Clause (Bulk Pattern)
Set<Id> accIds = new Set<Id>();
  • List conList = [
  • SELECT Id, Name
  • FROM Contact
  • WHERE AccountId IN :accIds
];
  • Critical for bulk-safe code.
Child to Parent
  • SELECT Name, Account.Name
  • FROM Contact
Parent to Child
SELECT Name, (SELECT Name FROM Contacts)
  • FROM Account
  • Subquery.
ORDER BY
  • SELECT Name
  • FROM Account
  • ORDER BY CreatedDate DESC
LIMIT & OFFSET
  • SELECT Name
  • FROM Account
  • LIMIT 5
OFFSET used for pagination (limited usage).
Aggregate SOQL
SELECT COUNT(Id) FROM Account
SELECT Industry, COUNT(Id)
  • FROM Account
  • GROUP BY Industry
  • Used for reporting logic.
üìå FOR UPDATE (Record Locking)
  • SELECT Id, Name
  • FROM Account
  • WHERE Id = :accId
  • FOR UPDATE
  • Locks record during transaction.
  • Prevents race conditions.
Governor Limits
  • Limit Type Sync Async
  • SOQL Queries 100 200
  • Rows Returned 50,000 50,000
  • Aggregate Queries 300 300
  • Exceeding ‚Üí LimitException.
üìå Query Selectivity (Very Important for Interview)
In Large Data Volume (LDV):
  • Query must:
  • Use indexed field
  • Return <10% of records
  • Avoid leading wildcard in LIKE
  • Avoid negative conditions
Bad ‚ùå
  • WHERE Name LIKE '%Tech%'
Good ‚úÖ
  • WHERE Industry = 'IT'
Query Plan Tool
  • Used to check:
  • Cost
  • Index usage
  • Cardinality
  • Selectivity
  • Very important in performance interviews.
Performance Best Practices
  • Query only required fields.
  • Use selective filters.
  • Avoid nested loops.
  • Use Maps.
  • Avoid full table scan.
  • Avoid querying in loops.
  • Use LIMIT if possible.
Too Many Query Rows: 50001
  • Use Batch Apex.
Non-Selective Query Against Large Object
  • Throws runtime exception.
Querying Large Text Area
  • Not filterable.
Debugging SOQL Issues
  • If performance issue:
  • 1Use Query Plan Tool.
  • 2Check debug logs.
  • 3Check filter selectivity.
  • 4Check indexed fields.
  • 5Check data skew.
When NOT to Use SOQL
  • Avoid using SOQL if:
  • Data already in Trigger.new.
  • Data available in Map.
  • Can use relationship reference.
Using Custom Metadata (cached better).
Interview Power Line
  • If interviewer asks:
  • ‚ÄúWhat is the biggest mistake developers make with SOQL?‚Äù
Answer:

Writing SOQL inside loops and not designing queries to be selective in large data volume environments.
