Future Methods

7.a Future Methods

üé§INTERVIEW SECTION
30-Second Interview Answer

Future Methods are asynchronous Apex methods that run in a separate thread after the current

transaction commits. They are defined using the @future annotation and are commonly used for callouts

  • and long-running operations that should not block the user transaction.
2-Minute Technical Explanation
  • Future methods:
  • Run asynchronously
  • Execute after database commit
  • Have separate governor limits
  • Cannot return values
  • Must be static
  • Accept only primitive types or collections of primitives
  • Basic syntax:
  • @future
public static void methodName(String param){
  • // async logic
}
  • Used when:
  • Performing HTTP callouts
  • Running long-running logic
  • Avoiding Mixed DML error
  • Preventing trigger CPU limit overflow

Why Future Method Exists
Because:
  • Synchronous transaction has CPU limit.
  • Callouts not allowed directly in trigger.
  • Mixed DML not allowed in same transaction.
  • Future provides:
  • Separate transaction
  • Separate limits
  • Post-commit execution

  • Must be static
  • Must return void
  • Cannot accept:
  • sObject
  • Objects
  • Complex types
  • No chaining
  • No guarantee of execution order

Performance Considerations
  • Good for:
  • Callouts
  • Light async work
  • Simple integration
  • Not ideal for:
  • Chaining
  • Complex logic
  • Retry handling
  • Large batch processing
  • Prefer Queueable instead.
When NOT to Use Future
  • Avoid if:
  • Need job chaining
  • Need job monitoring control
  • Need complex state handling
  • Need large data processing
Use:
  • Queueable Apex instead.
