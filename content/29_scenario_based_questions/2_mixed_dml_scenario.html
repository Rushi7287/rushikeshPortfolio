30-Second Interview Answer

A Mixed DML error occurs in Salesforce when a transaction attempts to perform DML operations on

setup objects (like User, Profile, PermissionSetAssignment) and non-setup objects (like Account,
Opportunity) in the same transaction. To fix it, we separate the operations using asynchronous Apex or
  • different transactions.

What Causes Mixed DML?
  • If transaction contains:
  • DML on Setup Object
  • +
  • DML on Non-Setup Object
Example:

insert acc;
User u = new User(...);
insert u;

  • ‚ÄúMIXED_DML_OPERATION‚Äù
Why Salesforce Restricts It
Because:
  • Setup object changes affect:
‚úî User permissions
‚úî Role hierarchy
‚úî Sharing recalculation
  • Mixing with business data DML can cause inconsistent state.
Fix Using Queueable
Trigger:
