30-Second Interview Answer

Dynamic SOQL is used when the SOQL query needs to be constructed at runtime, such as when object

names, field names, or filters are not known at compile time. It is executed using Database.query() and is

useful for flexible, metadata-driven applications. However, it must be handled carefully to prevent SOQL

  • injection.
2-Minute Technical Explanation

Security Risk ‚Äì SOQL Injection
  • Example Attack:
  • User enters:
  • IT' OR Name != ''
  • Query becomes:
  • WHERE Industry = 'IT' OR Name != ''
  • Returns all records.
Solution:
  • Use bind variables
  • Validate input
  • Escape single quotes
Performance Considerations
  • Dynamic SOQL:
  • Slightly slower than static
  • No compile-time validation
  • More risk of non-selective queries
  • Avoid using dynamic when static possible.
Governor Limits
  • Dynamic SOQL:
  • Counts toward 100/200 limit
  • Row limit still 50,000
  • Aggregate limit still applies
  • No special exemption.
Dynamic Field Selection Example

List<SObject> records = Database.query(query);

When NOT to Use Dynamic SOQL
  • Avoid if:
  • Query structure fixed.
  • Security risk high.
  • Performance critical.
  • Simple filtering needed.
  • Use static instead.
