üé§INTERVIEW SECTION
30-Second Interview Answer

Custom Labels are used to store reusable text values in Salesforce that can be referenced in Apex,

Visualforce, LWC, and validation rules. They support multilingual translations and help avoid hardcoding

  • user-facing text in code.
2-Minute Technical Explanation
  • Custom Labels are typically used for:
  • Error messages
  • UI messages
  • Integration messages
  • Constants displayed to users
  • Multilingual support
  • Created via:
  • Setup ‚Üí Custom Labels
  • Access in Apex:
String msg = Label.My_Error_Message;
Benefits:
  • Centralized text management
  • Supports translation
  • Avoids hardcoding
  • Easily editable without code change
Real-Time Scenario
Requirement:
  • Show validation message:
  • "Credit Limit Exceeded"
  • Instead of hardcoding:
acc.addError('Credit Limit Exceeded');
Use:
acc.addError(Label.Credit_Limit_Exceeded);
Now:
  • Admin can update text
  • Supports multiple languages
  • No deployment required

üî¨DEEP TECHNICAL BREAKDOWN
What is Custom Label?
  • A metadata-based text value.
  • Used for:
  • Messages
  • Titles
  • Descriptions
  • Integration constants
Creating Custom Label
Fields:
Name (API Name)
  • Value
  • Description
  • Language translations
Access in Apex
String errorMsg = Label.My_Custom_Label;
  • No SOQL required.
Access in LWC
Import:
import myLabel from '@salesforce/label/c.My_Custom_Label';
Use:
console.log(myLabel);
  • Clean UI integration.
Access in Visualforce
{!$Label.My_Custom_Label}
Multilingual Support
  • If user language = French
  • Salesforce automatically displays French translation.
  • No code change needed.
  • Enterprise-ready.
Using Label in Validation Rule
Example:
  • Error Message:
  • $Label.Credit_Limit_Error
  • Centralized management.
Best Practices
‚úî Use labels for all user-facing text
‚úî Use meaningful naming
‚úî Add description
‚úî Avoid storing logic values
‚úî Keep consistent naming convention
Custom Label vs Custom Metadata
  • Feature Custom Label CMDT
When NOT to Use Custom Label
  • Avoid for:
  • Numeric thresholds
  • Business rules
  • Feature flags
  • Integration endpoints
  • Use CMDT instead.
Enterprise Pattern
Trigger:
if(acc.Credit_Limit__c < acc.Total_Amount__c){
acc.addError(Label.Credit_Limit_Error);
}
Flow:
  • Uses same label.
UI:
  • Uses same label.
  • Single source of truth.
Performance Impact
  • No SOQL
  • Cached
  • Very lightweight
  • Safe for frequent usage.
Advanced Use Case
  • Store JSON template in Custom Label for integration payload skeleton.
  • But keep size reasonable.
