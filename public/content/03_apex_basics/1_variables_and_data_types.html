üé§INTERVIEW SECTION
30-Second Interview Answer

In Apex, variables are containers used to store data values, and because Apex is strongly typed, every

variable must be declared with a specific data type. Data types in Apex are categorized into primitive

types like Integer, Decimal, String, Boolean, Date, and non-primitive types like sObjects, Collections (List,
Set, Map), and custom classes.
2-Minute Technical Explanation
  • Apex is a strongly typed language, meaning:
  • Every variable must have a declared type.
  • Type cannot change at runtime.
  • Type safety is enforced at compile time.
  • Data types in Apex are divided into:
1. Primitive Data Types
  • Integer
  • Long
  • Decimal
  • Double
  • Boolean
  • String
  • Date
  • DateTime
  • Time
  • Id
  • Blob
2. Non-Primitive Data Types
  • sObject
Collections (List, Set, Map)
  • Enums
  • Apex Classes
  • Objects
Real-Time Scenario
Example:
  • When Opportunity is Closed Won:
Calculate commission (Decimal)
Check status (Boolean)
Store OwnerId (Id)
Update Account (sObject)
  • All require proper data types.
  • Wrong data type ‚Üí compilation error or runtime failure.
Tricky Interview Questions
  • 1Difference between Integer and Long?
  • Long stores larger numbers.
  • 2Difference between Decimal and Double?
Decimal is precise (financial use).
  • 3Is String immutable?
  • Yes.
  • 4Can we store null in primitive?
Yes (except some cases like Boolean in logic context).
  • 5What is Id data type?
  • Special type for Salesforce record IDs.

Performance Considerations
Bad:
  • Storing large data unnecessarily.
  • Using Double for currency.
  • Excessive string concatenation.
Good:
  • Use StringBuilder for large concatenation.
  • Clear unused collections.
  • Use appropriate data type.

Debugging Data Type Issues
  • If error occurs:
  • Check null values.
  • Check casting.
  • Check wrong data type usage.
  • Check heap usage.
When NOT to Use Certain Types
  • Avoid Double for currency.
  • Avoid String for numeric comparison.
  • Avoid List when Set sufficient.
  • Avoid Map if no key-based lookup needed.
