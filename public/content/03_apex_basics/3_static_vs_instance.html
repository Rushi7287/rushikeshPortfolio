Static Vs Instance

3.c Static vs Instance

Next

üé§INTERVIEW SECTION
30-Second Interview Answer

In Apex, a static member belongs to the class itself and is shared across all instances within a

transaction, while an instance member belongs to a specific object created from the class. Static

  • variables are commonly used for utility methods and recursion control in triggers.
Static
  • Belongs to class, not object.
  • Accessed without creating object.
  • Shared across entire transaction.
  • Commonly used for:
  • Utility methods
  • Constants
  • Recursion control
  • Caching data
Example:
public class Utility {
public static void greet() {
System.debug('Hello');
}
}
  • Call directly:
Utility.greet();
Instance
  • Requires object creation.
  • Each object has its own copy of variables.
Example:
public class Person {
public String name;
}
Usage:
Person p1 = new Person();
p1.name = 'Rushikesh';
Recursion Handling in Trigger
public class TriggerHelper {
public static Boolean isFirstRun = true;
}
  • In trigger:
if(TriggerHelper.isFirstRun){
TriggerHelper.isFirstRun = false;
}
  • Prevents infinite loop.
Tricky Interview Questions
  • 1Do static variables persist across transactions?
‚ùå No (reset per transaction).
  • 2Can static constructor exist?
‚ùå No (Apex doesn‚Äôt support static blocks like Java).
  • 3Can static method access instance variable?
‚ùå No (without object).
  • 4Can instance method access static variable?
Yes.
  • 5Does static reset in Batch Apex?
‚úÖ Yes (each batch execution is new transaction).

  • Static in Queueable chaining
  • Static cache pattern

Performance Considerations
Good:
  • Use static for caching.
  • Use static for constants.
  • Use static for recursion control.
Bad:
  • Using static for cross-transaction storage.
  • Using static for user session memory.
Static in Test Class
  • Static resets between test methods.
Static in Queueable Chaining
  • Does not carry forward to next Queueable.
Static & Parallel Execution
  • Each transaction isolated.
Debugging Static Issues

  • 1Check if transaction restarted.

  • 4Check order of execution.
When NOT to Use Static
  • For storing user-specific data.
For persistent configuration (use Custom Metadata).
  • For cross-transaction tracking.

üî∑ 3.d Collections (List, Set, Map)
üé§INTERVIEW SECTION
30-Second Interview Answer

Collections in Apex are used to store and manipulate multiple records efficiently. The three main types

are List (ordered, allows duplicates), Set (unordered, unique values only), and Map (key-value pair
structure). Collections are critical for bulkification and preventing governor limit issues.
2-Minute Technical Explanation
  • Collections are essential in Apex because:
  • Triggers handle multiple records.
  • Governor limits restrict SOQL and DML.
  • Bulk operations require grouping data.
List
  • Ordered
  • Allows duplicates
Indexed (0-based)
Set
  • Unordered
  • No duplicates
  • Faster lookup than List
Map
  • Key ‚Üí Value structure
  • Fast retrieval using key
  • Most powerful for bulk processing
Real-Time Scenario
Requirement:
  • When Accounts updated,
  • Update all related Contacts.
Bad approach ‚ùå:
  • Query inside loop.
Good approach ‚úÖ:
  • 1Collect Account Ids in Set.
  • 2Query Contacts using IN clause.
  • 3Store in Map.
  • 4Update in bulk.
