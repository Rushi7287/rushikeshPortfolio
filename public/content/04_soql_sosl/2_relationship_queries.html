30-Second Interview Answer

Relationship Queries in SOQL allow retrieving related records using parent-to-child subqueries or child-

to-parent dot notation. Child-to-parent uses dot notation like Account.Name, while parent-to-child uses a

nested subquery like (SELECT Name FROM Contacts). These queries help reduce multiple SOQL calls and
  • improve performance.
2-Minute Technical Explanation
  • Salesforce relationships:
  • Lookup
  • Master-Detail
  • SOQL allows navigating them in two ways:
üîπ Ûæ†Æ Child ‚Üí Parent (Many-to-One)
  • Using dot notation:
  • SELECT Name, Account.Name
  • FROM Contact
  • You move upward in hierarchy.
Rule:
  • Max 5 levels upward.
üîπ Ûæ†Ø Parent ‚Üí Child (One-to-Many)
  • Using subquery:
SELECT Name, (SELECT Name FROM Contacts)
  • FROM Account
  • You move downward.
Rule:
  • Only 1 level down per query.
Real-Time Scenario
Requirement:
  • When Account updated:
  • Update all related Contacts.
  • Instead of 2 queries:
  • Use parent-to-child relationship query.
  • Saves governor limits.

üìå Semi-Join (Advanced)
  • Query parent based on child condition.
  • SELECT Name
  • FROM Account
WHERE Id IN (
  • SELECT AccountId
  • FROM Contact
  • WHERE LastName = 'Sharma'
)
Anti-Join
  • Query parent without matching child.
  • SELECT Name
  • FROM Account
WHERE Id NOT IN (
  • SELECT AccountId FROM Contact
)
Governor Limit Behavior
  • Relationship query counts as:
  • 1 SOQL
  • Child rows count toward 50,000 row limit
  • Important for large data.
Performance Considerations
Good:
  • Avoid querying unnecessary child records.
  • Use filters inside subquery.
  • Use selective WHERE clause.
Bad:
  • Fetching thousands of child records.
  • Deep nested loops after subquery.
Heap Risk Example
  • If 100 Accounts
  • Each has 1,000 Contacts
  • Total rows = 100,000
‚Üí May hit heap size or row limit.
  • Use batch in such case.
Wrong Child Relationship Name
  • QueryException thrown.
No Child Records
  • Subquery returns empty list.
Large Text Area in Subquery
  • May increase heap usage.
Debugging Relationship Query Issues
  • If query fails:
  • 1Check relationship name.
  • 2Check object relationship type.
  • 3Check field-level security.
  • 4Check row limits.
  • 5Check selectivity.
When NOT to Use Relationship Query
  • Avoid if:
  • Very large child data.
  • Only few child records needed across many parents.
  • Better to query child separately and map manually.

  • ‚ÄúWhat is difference between child-to-parent and parent-to-child query?‚Äù

  • a nested subquery and retrieves related child records in a single SOQL statement.
