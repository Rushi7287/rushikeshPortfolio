30-Second Interview Answer
SOSL (Salesforce Object Search Language) is used to perform text-based searches across multiple

objects and fields simultaneously. Unlike SOQL, which queries a single object, SOSL searches across

multiple objects and returns results in a list of lists format. It is optimized for full-text search.

2-Minute Technical Explanation
  • Difference between SOQL and SOSL:
SOQL ‚Üí Structured query (like SQL), one object at a time.
  • SOSL ‚Üí Text search across multiple objects.
  • Use SOSL when:
  • Searching by keyword.
  • Searching across multiple objects.
  • Searching unknown field location.
  • Implementing global search feature.
  • Basic Syntax:
  • FIND 'searchTerm'
  • IN ALL FIELDS
RETURNING Account(Id, Name), Contact(Id, Name)
Returns:
  • List>
  • Each list corresponds to one object in RETURNING clause.
Real-Time Scenario
Requirement:
  • User types:
  • "Tech"
  • System must search:
  • Account Name
  • Contact Name
  • Opportunity Name
  • Instead of writing 3 SOQL queries:
  • Use single SOSL query.

Basic SOSL Syntax

  • List> results =
  • [FIND 'John'

Accessing SOSL Results
List<Account> accList = (List<Account>) results[0];
List<Contact> conList = (List<Contact>) results[1];
  • Index order matches RETURNING order.
Search Scope Options
  • IN ALL FIELDS
  • IN NAME FIELDS
  • IN EMAIL FIELDS
  • IN PHONE FIELDS
Example:
FIND '9876543210' IN PHONE FIELDS RETURNING Contact(Id, Name)
Filtering in RETURNING Clause
  • FIND 'Tech'
RETURNING Account(Id, Name WHERE Industry = 'IT' LIMIT 5)
  • Limited filtering allowed inside RETURNING.
Governor Limits
  • Limit Type Value
  • SOSL Queries 20 per transaction
  • Rows Returned 2,000 per object
Important:
  • SOSL has lower limit than SOQL.

Performance Considerations
  • Use SOSL when:
  • Searching unknown location.
  • Keyword-based search.
  • Cross-object search.
  • Use SOQL when:
  • Structured filtering.
  • Known object.
  • Relationship-based query.
Indexing Behavior
  • SOSL uses:
  • Search index
  • Full-text index
  • Faster for:
  • Large text search
  • Global search features
Newly Created Record Not Appearing
  • Search index delay may cause slight lag.
Non-searchable Fields
  • Not all fields searchable.
Large Result Sets
  • Limited to 2,000 rows per object.
Debugging SOSL Issues
  • If no results:
  • 1Check search term.
  • 2Check field searchable.
  • 3Check object included in RETURNING.
  • 4Check user access.
  • 5Check indexing delay.
When NOT to Use SOSL
  • Avoid when:
  • Filtering by structured criteria.
  • Need exact relationship query.
  • Only querying one object with known filters.
  • Prefer SOQL.
