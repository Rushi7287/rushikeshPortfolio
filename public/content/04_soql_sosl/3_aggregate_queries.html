üé§INTERVIEW SECTION
30-Second Interview Answer

Aggregate Queries in SOQL are used to perform calculations on records, such as COUNT, SUM, AVG,

MIN, and MAX. They are commonly used with GROUP BY to summarize data, similar to SQL aggregation.

  • The results are returned as AggregateResult objects.
2-Minute Technical Explanation
  • Aggregate SOQL allows:
  • Counting records
  • Summing numeric fields
  • Grouping by field values
  • Filtering grouped results using HAVING
Example:
SELECT Industry, COUNT(Id)
  • FROM Account
  • GROUP BY Industry
Returns:
  • Each Industry
  • Number of Accounts per Industry
  • Returned as:
  • List
Real-Time Scenario
Requirement:
  • Show total revenue per Account.
  • Instead of looping through Opportunities manually:
  • Use aggregate query:
SELECT AccountId, SUM(Amount)
  • FROM Opportunity
  • WHERE IsClosed = true
  • GROUP BY AccountId
  • Much faster and limit efficient.

  • GROUP BY Industry

Important:
  • Use alias to retrieve values.
GROUP BY Multiple Fields
SELECT AccountId, StageName, SUM(Amount)
  • FROM Opportunity
  • GROUP BY AccountId, StageName
HAVING Clause
  • Filter grouped result.
SELECT AccountId, SUM(Amount)
  • FROM Opportunity
  • GROUP BY AccountId
HAVING SUM(Amount) > 100000
üìå GROUP BY ROLLUP (Advanced)
  • Provides subtotal and grand total.
SELECT AccountId, SUM(Amount)
  • FROM Opportunity
GROUP BY ROLLUP(AccountId)
  • Used in reporting-like scenarios.
Governor Limits
  • Limit Type Value
  • Aggregate Queries 300
  • Rows Returned 50,000
Aggregate queries still count toward SOQL limit (100/200).
Performance Benefits
  • Aggregate SOQL:
  • Executes at database level.
  • Reduces Apex CPU time.
  • Avoids manual loops.
  • More scalable.
  • Better than:
  • Looping through thousands of records to calculate sum.
Roll-Up Summary vs Aggregate SOQL

Debugging Aggregate Issues
  • If error occurs:
  • 1Check GROUP BY fields.
  • 2Check alias usage.
  • 3Check casting.
  • 4Check row limit.
  • 5Check data skew.
When NOT to Use Aggregate
  • Avoid if:
  • You need individual record processing.
  • Complex per-record logic required.
  • Large heap due to many grouped results.
  • Use Batch Apex instead.
