30-Second Interview Answer

The Complete Save Order is the exact sequence Salesforce follows when a record is created or updated.

It includes system validation, before triggers, custom validation rules, after triggers, workflow rules,

flows, roll-up calculations, commit, and post-commit operations. Understanding this helps prevent

  • recursion and unexpected data changes.
2-Minute Technical Explanation

When a record is saved, Salesforce executes automation in a strict sequence within a single transaction.

  • High-Level Complete Order:
  • 1Load Existing Record (if update)
  • 2System Validation
  • 3Before Triggers
  • 4Custom Validation Rules
  • 5Duplicate Rules
  • 6Save Record (not committed yet)
  • 7After Triggers
  • 8Assignment Rules
  • 9Auto-response Rules
  • 10Workflow Rules
  • 11Workflow Field Updates (may cause re-entry)
  • 12Process Builder / After-Save Flow
  • 13Roll-Up Summary Recalculation
  • 14Criteria-Based Sharing
  • 15Commit to Database
  • 16Post-Commit Logic (Async, Email, Events)
  • Every step shares same governor limits until commit.
Real-Time Scenario
  • You update Account:
  • Before trigger modifies field.
  • Validation rule checks field.
  • After trigger creates Contact.
  • Workflow updates Rating.
  • Trigger fires again.
  • After-save Flow sends email.
  • Record commits.
  • Queueable job executes.
