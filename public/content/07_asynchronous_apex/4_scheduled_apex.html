üé§INTERVIEW SECTION
30-Second Interview Answer

Scheduled Apex is used to execute Apex code at a specific time or on a recurring schedule. It

implements the Schedulable interface and is commonly used for nightly jobs, periodic cleanups, report

  • generation, or launching batch processes automatically.
2-Minute Technical Explanation
  • Scheduled Apex allows you to:
  • Run Apex at fixed intervals
  • Automate recurring processes
  • Trigger Batch Apex automatically
  • Perform system maintenance tasks
  • Basic structure:
public class MyScheduler implements Schedulable {
public void execute(SchedulableContext sc) {
  • // logic here
}
}
  • Schedule it:
String cronExp = '0 0 2 * * ?'; // 2 AM daily
System.schedule('Nightly Job', cronExp, new MyScheduler());
  • Typically used to start Batch Apex:
Database.executeBatch(new MyBatch());
Real-Time Scenario
Requirement:
  • Every night at 2 AM:
  • Recalculate commissions
  • Send summary report
  • Sync data with ERP
Solution:
  • Scheduled Apex triggers Batch Apex at 2 AM daily.

Cron Expression Breakdown
Format:
Seconds Minutes Hours Day_of_month Month Day_of_week Year(optional)
Example:
  • 0 0 2 * * ?
Meaning:
  • 0 seconds
  • 0 minutes
  • 2 AM
  • Every day
  • Every month
  • Any day of week

üìå Scheduler + Batch Pattern (Best Practice)
  • Instead of heavy logic in scheduler:
public void execute(SchedulableContext sc){
Database.executeBatch(new MyBatch(), 200);
}
  • Scheduler only starts batch.
  • Batch handles heavy processing.
Governor Limits
  • Scheduled Apex:
  • Runs in its own transaction
Same limits as async (200 SOQL, 60 sec CPU)
  • Each scheduled execution new transaction

Avoid Overlapping Execution
Problem:
  • If job takes 2 hours
  • But scheduled every 1 hour
‚Üí Overlap occurs.
Solution:
  • Check AsyncApexJob status
  • Prevent duplicate scheduling

Use:

Test.stopTest();

Sandbox Refresh
  • Scheduled jobs may be removed or paused.
When NOT to Use Scheduled Apex
  • Avoid if:
  • One-time execution ‚Üí Use anonymous Apex
  • Immediate async ‚Üí Use Queueable
  • Large recurring data ‚Üí Use Scheduler + Batch
