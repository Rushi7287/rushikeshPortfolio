30-Second Interview Answer

Batch Apex is used to process large volumes of records asynchronously by breaking them into

manageable chunks called batches. It implements the Database.Batchable interface and processes

  • records in separate transactions, resetting governor limits for each batch execution.
2-Minute Technical Explanation
  • Batch Apex is designed for:
Large Data Volume (LDV)
  • Processing more than 50,000 records
  • Scheduled bulk processing
  • Data cleanup jobs
  • Recalculations

Real-Time Scenario
Requirement:

Why Batch Apex Is Needed
Because:
  • SOQL limit: 50,000 rows in sync
  • Heap size limit
  • CPU limit
  • Need to process millions of records
  • Batch splits processing into chunks.

Governor Limits Per Batch
  • Each execute:
  • 200 SOQL
  • 150 DML
  • 60 sec CPU
  • 12MB heap
  • Resets for next chunk.
  • Huge advantage.
Batch Size Tuning
Default: 200
  • Smaller batch size:
  • Less heap usage
  • Lower CPU per chunk
  • Larger batch size:
  • Fewer transactions
  • Higher per-chunk resource use
  • Common tuning:
  • 50‚Äì200 depending on complexity.

Error Handling Strategy

  • Use try-catch.
  • Log failed records in custom object.
  • Avoid failing entire batch.

When NOT to Use Batch
  • Avoid if:

  • Immediate response needed
  • Simple async logic
  • Use Queueable instead.
