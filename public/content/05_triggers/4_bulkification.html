30-Second Interview Answer

Bulkification is the practice of writing Apex code that can handle multiple records efficiently in a single

transaction. Since triggers and API operations can process up to 200 records at once, code must avoid

SOQL or DML inside loops and use collections like Set and Map to stay within governor limits.

2-Minute Technical Explanation
  • In Salesforce:
  • A single trigger execution can process up to 200 records.
  • Data Loader or integrations can send records in bulk.
  • Governor limits restrict:
  • 100 SOQL queries
  • 150 DML statements
  • If code is not bulkified:
  • It will fail in production.
  • Bulkification principles:
  • Ûæ†Æ Never write SOQL inside loop

  • Ûæ†∞ Use Set to collect Ids
  • Ûæ†± Use Map for fast lookup

Why Bulkification Is Required
  • Trigger may execute for:
UI save (1 record)
Data Loader (200 records)
Integration API (200 records)
  • Same code must handle all scenarios.
Core Bulkification Pattern
  • Step 1: Collect IDs
Set<Id> accIds = new Set<Id>();
  • Step 2: Query Once
Map<Id, Account> accMap = new Map<Id, Account>(
  • [SELECT Id, Name FROM Account WHERE Id IN :accIds]
);
  • Step 3: Process in Loop
for(Contact con : conList){
Account acc = accMap.get(con.AccountId);
}
  • Step 4: Single DML
update conList;
üìå Advanced Bulk Pattern (Grouping)
Example:

Map<Id, List<Contact>> accToContacts = new Map<Id, List<Contact>>();
for(Contact con : conList){
if(!accToContacts.containsKey(con.AccountId)){
accToContacts.put(con.AccountId, new List<Contact>());
}
accToContacts.get(con.AccountId).add(con);
}
  • Used in complex logic.

Governor Limit Awareness
  • Bulkified code:
  • Minimizes SOQL usage.
  • Minimizes DML usage.
  • Reduces CPU time.
  • Avoids heap overflow.
