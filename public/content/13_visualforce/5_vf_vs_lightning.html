14.a Record-Triggered Flow (Before-Save & After-Save)
üé§INTERVIEW SECTION
30-Second Interview Answer
  • Record-Triggered Flow automatically runs when a record is created, updated, or deleted.

Before-Save Flow is used to update fields efficiently before the record is committed to the

database (faster, no extra DML).

After-Save Flow runs after the record is saved and is used for related record creation, callouts, or

  • complex logic.
2-Minute Technical Explanation
  • Record-Triggered Flow is similar to Apex Trigger but declarative.
  • Two types:
Before-Save Flow
  • Runs before database commit
  • Can update triggering record fields
  • Cannot perform DML on other records
  • Cannot call subflow with DML
  • 10x faster than after-save
  • Best for field updates
  • Equivalent to:
  • before insert, before update
After-Save Flow
  • Runs after record committed
  • Can create/update related records
  • Can call subflows
Can perform callouts (via async path)
  • Slightly slower
  • Equivalent to:
  • after insert, after update
Real-Time Scenarios
  • Scenario 1
  • If Amount > 10000 ‚Üí Set Priority = High
‚úî Use Before-Save Flow
  • Scenario 2
  • When Opportunity Closed Won ‚Üí
  • Create Commission Record
‚úî Use After-Save Flow
  • Scenario 3
  • When Account created ‚Üí
  • Send data to external system
‚úî After-Save + Async Path
