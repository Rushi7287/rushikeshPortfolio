üé§INTERVIEW SECTION
30-Second Interview Answer
  • In LWC:
  • @api is used to expose public properties or methods to parent components.
@track (legacy) was used to make properties reactive, but now objects and arrays are reactive by
  • default.
@wire is used to connect LWC to Salesforce data (Apex or LDS) reactively.
api
  • Used for:
  • Public properties
  • Public methods
  • Parent ‚Üí Child communication
Example:
import { LightningElement, api } from 'lwc';
export default class Child extends LightningElement {
@api recordId;
}
  • Parent passes value to child.
üîπ @track (Important Update)
Earlier:
  • Used for object/array reactivity.
Now:
  • Primitive, object, and array properties are reactive by default.
  • Only needed in rare deep mutation cases.
wire
  • Used for:
  • Calling Apex
  • Getting record data
  • Fetching metadata
  • Reactive and auto-refreshing.
@wire(getAccount, { accId: '$recordId' })
account;
  • When recordId changes ‚Üí wire re-executes.
Real-Time Scenario
Requirement:
  • Display Account data on record page.
  • @api recordId receives record ID
  • @wire fetches Account
  • UI auto-updates when record changes
  • Clean reactive architecture.

üìå Reactive Parameter ($ Syntax)

Wire vs Imperative Call
  • Feature @wire Imperative
  • Reactive Yes No
  • Auto Refresh Yes No
  • Cacheable Required Yes No
  • Control Less More
  • Use Case Display data On button click
Performance Consideration
  • Best practice:
‚úî Use LDS or @wire for read
‚úî Use imperative for create/update
‚úî Avoid unnecessary reactive parameters
Common Bug Scenario
  • Developer forgets $:
@wire(getAccount, { accId: 'recordId' })
  • Wire never re-executes when recordId changes.
  • Must use:
  • '$recordId'
Enterprise Pattern
  • Parent LWC:
@api recordId;
  • Child LWC:
  • Receives via @api
  • Calls Apex via @wire
  • Displays data
  • Clear separation of responsibilities.
Best Practices Summary
‚úî Use @api for parent-child communication
‚úî Avoid unnecessary @track
‚úî Use $ for reactive wire params
‚úî Prefer LDS when possible
‚úî Use function wire for error handling
