30-Second Interview Answer

Skinny Tables in Salesforce are custom database tables maintained by Salesforce Support that contain a

subset of frequently used fields from a large object to improve query performance in high data volume

  • environments.
2-Minute Technical Explanation
In LDV orgs (millions of records), performance issues occur when:
  • Object has many fields
  • Frequent queries retrieve same subset of fields
  • Joins across multiple tables slow down execution
  • Skinny Tables solve this by:
‚úî Creating a physical table in Salesforce database
‚úî Copying selected frequently used fields
‚úî Reducing table joins
‚úî Speeding up SELECT queries
Important:
  • They are:
  • Created by Salesforce Support only
  • Transparent to developers
  • Used automatically by query optimizer
Real-Time Scenario
  • Org has:
  • 20 million Account records
  • 200+ fields
  • Reports always query:
  • Name
  • Region__c
  • Industry
  • CreatedDate
  • Performance slow.
Solution:
  • Request Skinny Table with those 4 fields.
Now:
  • Query performance improves significantly.
